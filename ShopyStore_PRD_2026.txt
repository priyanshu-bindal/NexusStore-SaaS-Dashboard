Executive Summary

ShopyStore is a multi-tenant SaaS storefront and seller dashboard built
on a strong modern foundation — React 19, Next.js 15, Prisma, and
PostgreSQL. However, the audit reveals a cluster of high-severity
technical debts that are actively hurting performance, SEO visibility,
and accessibility. This PRD defines the complete remediation plan, 2026
roadmap additions, and a list of legacy patterns to retire.

  ------------- ------------------- -------------------------------------
  Severity      Finding             Detail

  🔴 Critical   Image Optimization  6+ views use raw <img> tags — no
                                    WebP/AVIF compression, no lazy
                                    loading, guaranteed CLS

  🔴 Critical   SEO Infrastructure  No sitemap.xml, no robots.txt, 24 of
                Missing             26 pages lack unique <title> or OG
                                    metadata

  🔴 Critical   Missing 404 Handler No not-found.tsx on dynamic routes —
                                    invalid product URLs crash to blank
                                    screens

  🟡 Warning    Accessibility       Only ~5 aria- labels codebase-wide —
                (A11y)              modals, toasts and dropdowns trap
                                    screen readers

  🟡 Warning    Heavy Client        Recharts and Framer Motion imported
                Bundles             directly with no dynamic() lazy
                                    splitting

  🟡 Warning    No DB Connection    Prisma on Serverless without
                Pooling             PgBouncer/Accelerate risks exhausting
                                    connection limits

  🟢 Minor      Strong Architecture React 19 + Next.js 15 + Server
                Base                Actions + Zod — solid, scalable,
                                    future-proof foundation

Tab Switching & Page Visibility

These specific patterns cause the 'things loading again' and 'shows
twice' bugs on tab switch:

-   Root Cause 1: Framer Motion AnimatePresence keys — missing or
    duplicated keys cause components to remount on visibility change,
    appearing to render twice. Fix: audit every AnimatePresence and
    ensure every child has a stable, unique key.

-   Root Cause 2: SWR / React Query without focus revalidation config —
    by default, these refetch on window focus. Add refetchOnWindowFocus:
    false to queries that don't need it (analytics, product lists). Keep
    it enabled only for cart and order status.

-   Root Cause 3: Polling intervals that don't pause — any setInterval
    fetch should be paused using the Page Visibility API. Add:
    document.addEventListener('visibilitychange', () => { if
    (document.hidden) clearInterval(poll); else startPolling(); })

-   Root Cause 4: Animation restarts — CSS animations with
    animation-fill-mode: none restart on tab return. Add
    animation-fill-mode: forwards to all non-looping animations.
